# Thread Rules

## Core Rules
- One thread per workstream.
- No interface changes after Phase 2 without updating `DECISIONS.md`.
- Every thread ends with a written handoff.
- Admin handoff notes live in `00_admin/codex_threads/handoffs/`.
- Closed-thread summaries live in `00_admin/codex_threads/archive/`.

## Working Discipline
- Keep file ownership clear before editing.
- Keep changes small and scoped to the thread goal.
- If a thread needs to touch a new area, update `00_admin/CODEX_THREAD_MAP.md` first.
- Every material change must update `CHANGELOG.md`.
- Any direction change must update `DECISIONS.md`.
- Every verification statement must say whether it is locally verified or only assumed.
- Detailed phase notes may also live in `04_execution/`, but the admin handoff path must still be recorded in `00_admin/codex_threads/handoffs/`.
